API Design Review — Transaction.java (public interface)
MODEL=opus-4.8  DATE=2026-06-27
RE-VERIFIED 2026-06-27 against current source (line numbers refreshed)

1. OVERALL ASSESSMENT
Transaction is a small, clean contract: id() (46), isolationLevel() (72), status() (90), isActive() (110),
commit() (138), rollback() (165), rollbackIfNotCommitted() (197), plus nested enums Status (211) and
Action (275). The accessor names follow the library's fluent no-"get" convention and are consistent with the
SqlTransaction implementation. The only actionable issue is a cross-type deprecation/consistency gap around
rollback().

2. NAMING ISSUES
- None.

3. RETURN-TYPE AND PARAMETER ISSUES
- None. All methods return appropriately (String/IsolationLevel/Status/boolean/void) and are documented
  "never null" where relevant.

4. MISPLACED METHODS
- None.

5. MISSING METHODS (with target class)
- None required. (The concrete SqlTransaction adds connection()/runOutsideTransaction/callOutsideTransaction;
  these are deliberately NOT on the interface and callers reach them via the concrete return type of
  JdbcUtil.beginTransaction — re-verified acceptable, no public API returns the bare Transaction type. See
  SqlTransaction report.)

6. DEPRECATION / REMOVAL CANDIDATES (with replacement APIs)
- STILL-VALID — WON'T FIX (user decision 2026-06-27: interface rollback() will NOT be @Deprecated) | rollback()
  | Transaction.java:165 | Medium | The implementation SqlTransaction @Deprecates
  its rollback() in favor of rollbackIfNotCommitted() (SqlTransaction.java:472-474), and this interface's own
  Javadoc notes the deprecation in prose (Transaction.java:158-159: "the {@link SqlTransaction}
  implementation deprecates direct calls to this method in favor of {@link #rollbackIfNotCommitted()}") — yet
  the interface method itself is STILL NOT marked @Deprecated. Current: `void rollback() throws
  UncheckedSQLException;` (165) — no @Deprecated annotation. A caller holding a Transaction reference
  therefore sees no compiler deprecation signal. Recommend: mark Transaction.rollback() @Deprecated
  (replacement: rollbackIfNotCommitted()) for cross-type consistency. Confirming check: re-verified no
  implementation/caller depends on a non-deprecated rollback() on the interface (only SqlTransaction
  implements it; production code calls Connection.rollback(), not Transaction.rollback()).
- STILL-VALID — WON'T FIX (user decision 2026-06-27: example left as-is, tied to the rollback() deprecation
  above) | Action-enum Javadoc example calls transaction.rollback() | Transaction.java:270 | Low | The
  nested Action-enum usage example STILL calls the (implementation-)deprecated method. Current line 270:
  `*     transaction.rollback();` (inside the example at 265-272). Recommend: update the example to
  rollbackIfNotCommitted() to stop demonstrating the deprecated path. (Same applies to the rollback() method's
  own example at Transaction.java:153, which also shows `transaction.rollback();`.)
