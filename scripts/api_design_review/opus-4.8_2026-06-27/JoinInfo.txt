1. OVERALL ASSESSMENT
JoinInfo is annotated @Internal (line 115) — a low-level metadata/SQL-fragment holder used by the DAO
join-entity helpers; most logic lives in the constructor and private helpers. The public surface is small and
largely healthy: the three static factories (getEntityJoinInfo:1198, getPropJoinInfo:1267,
getJoinEntityPropNamesByType:1329) have consistent (daoClass, entityClass, tableName, ...) ordering and caching;
the four "SQL plan" getter families (getSelectSqlPlan, getBatchSelectSqlPlan, getDeleteSqlPlan,
getBatchDeleteSqlPlan) follow a consistent naming/return shape and each still has a deprecated
*SqlBuilderAndParamSetter predecessor with a clean one-line redirect (a well-managed rename). NET CHANGE since the
round-1 report: NONE — the file was untouched in the latest commit round; all cited lines re-confirmed exact. The
prior Javadoc fix that corrected the misleading @param sbc wording (now "the SQL builder DSL to use", e.g. line
745) remains in place, so the naming finding stays PARTIALLY-ADDRESSED. All other findings remain STILL-VALID; no
finding became obsolete and no NEW design defect was introduced.
