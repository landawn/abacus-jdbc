1. OVERALL ASSESSMENT
JoinInfo is annotated @Internal (line 115) — it is a low-level metadata/SQL-fragment holder used by the
DAO join-entity helpers, and most of its logic lives in the constructor and private helpers. The public
surface is small and largely healthy: the three static factories (getEntityJoinInfo, getPropJoinInfo,
getJoinEntityPropNamesByType, now ~1198/1267/1329) have consistent (daoClass, entityClass, tableName, ...)
ordering and caching; the four "SQL plan" getter families (getSelectSqlPlan, getBatchSelectSqlPlan,
getDeleteSqlPlan, getBatchDeleteSqlPlan) follow a consistent naming/return shape and each still has a
deprecated *SqlBuilderAndParamSetter predecessor with a clean redirect (a well-managed rename). Because
the class is @Internal, this report is kept proportionate. NET CHANGE since the prior report: the Javadoc
fix pass corrected the misleading @param sbc wording (was "the SQL builder class type"; now "the SQL builder
DSL to use") — so the naming finding is now PARTIALLY-ADDRESSED (wording fixed, parameter name `sbc` kept).
All other findings remain STILL-VALID; no findings became obsolete and no NEW design defects were introduced.

  | Medium (Low priority: class is @Internal) | The parameter is STILL named `sbc` (legacy "SqlBuilder

4. MISPLACED METHODS
- [STILL-VALID / None] Instance SQL-plan getters and setJoinPropEntities are correctly tied to
  per-relationship metadata; the static factories are the correct entry points. Nothing belongs on JdbcUtil
  or a DAO helper. No change.

5. MISSING METHODS (with target class)

  caches setNullSqlAndParamSetterPool (the UPDATE ... SET referenced-FK = default plan), paralleling the
  delete/select pools — but unlike all four sibling pools it has no public getter and is never read anywhere.
  EVIDENCE (grep): only 3 hits in the file — declaration `private final Map<Dsl, Tuple2<String,

  (line 150) and two `.put(...)` (lines 481, 659); zero `get`/getter references; no getSetNullSqlPlan symbol

  getter mirroring getDeleteSqlPlan, OR delete the pool and its two population blocks (and the now-unused
